## Documentation Rules

- Keep the root README short enough to remain a useful front door.
- Put ownership and policy in the responsible department.
- Put operating procedures next to the service or deployment they govern.
- Use relative links for repository documents and tested commands for examples.
- Do not paste credentials, private data samples, or live factor code into docs.
